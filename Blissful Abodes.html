BLISSFUL ABODES CASE STUDY

Project Overview

Blissful Abodes is a next-generation web application designed to handle end-to-end hotel bookings, cancellations, invoices, dynamic room recommendations, and automated risk checkins. Built initially as a local Flask booking application, this project showcases the complete re-architecture of the system into the AWS cloud environment.

Who it is for: Mid-to-large-scale hospitality managers seeking to reduce administrative workloads and guests who want a personalized, self-service checking flow with zero friction.

Main Objective: To build a scalable, highly secure infrastructure that integrates predictive AI engines directly into a traditional CRUD booking platform.

The Problem Statement

Legacy hospitality software suffers from severe architectural bottlenecks and vulnerabilities:

  • Business Threat (Fraud): High rates of booking scams and false refund claims cost operations significant manual audit hours.
  • User Pain Points: Long wait times at checking desks and static, un-optimized room pricing models that fail to match seasonal demands.
  • Technical Debt: A monolithic, single-database construct prone to outages during seasonal spikes, with local session state blocking horizontal scaling.
Engineered Solution

The Cloud-Native Solution

The application was re-architected on AWS, decoupling data layers, migrating file uploads to distributed object storage, and implementing standard enterprise authentication. Sessions and chatbot conversation logs are offloaded to DynamoDB, while relational datasets are housed in a Postgres database hosted on RDS.

Five intelligent ML models (using Scikit-Learn) run alongside the core Flask thread, intercepting bookings to analyze fraud likelihood, forecast cancellation probabilities, evaluate guest feedback sentiments, and dynamically set optimized room tariffs.

IAM Instance Profile Roles

Replaced hardcoded access keys on EC2 with strict role-based AWS policy checks.

Zero-State Application Nodes

Offloaded session state and chatbot histories to DynamoDB with custom TTLs.

Asynchronous Decoupling

Notifications and high-priority fraud alerts are pushed to AWS SNS topics.

AWS Infrastructure

AWS Service Integrations

Amazon EC2

Compute

Hosts the production Flask app inside an Ubuntu instance. Configured with a systemd daemon file, Gunicorn handlers, and Nginx acting as a reverse proxy on ports 80/443.

Instance Type t2.micro

λ AWS Lambda

Serverless

Executes serverless reporting functions. When guests complete their bookings, Lambda triggers asynchronously to generate the PDF receipt, avoiding CPU spikes on the main thread.

Execution Time < 350ms

Amazon S3

Storage

Acts as durable object storage for high-resolution hotel room images, guest profile uploads, and compiled PDF checkout reports. Configured with a lifecycle policy.

Durability 99.999999999%

AWS IAM

Security

Enforces the principle of least privilege. The EC2 instances are attached to an IAM Instance Profile, obtaining STS temporary session credentials instead of saving credentials on disk.

Access Model Role-Based (STS)

CloudWatch

Monitoring

Aggregates Nginx access logs and custom application metrics. CPU metrics are tracked; if threshold limit is hit during ML inference, an SNS alarm fires an alert.

Alarm Threshold 80% CPU

DynamoDB

NoSQL DB

Stores session states, user audit trails, and conversational histories from the Aria AI Concierge. Dynamically purges stale entries using native Time-To-Live (TTL) timestamps.

Latency < 10ms

AWS VPC

Networking

Isolates database layers from the public web. Relational RDS databases run in private subnets, while EC2 hosts are located in public subnets routed via an Internet Gateway.

Subnets Config Public / Private

Amazon RDS

Relational DB

Hosts production PostgreSQL. Manages booking details, room structures, rates, and guest feedback records. Automatic nightly backups and point-in-time recovery are active.

Engine PostgreSQL

Amazon SNS

Messaging

Publishes booking transactional confirmations to guests and pushes immediate warning notifications to administrator mobile targets when high-risk bookings are flagged.

Endpoints Email & SMS

Amazon SQS

Queueing

Decouples email queues from the booking checkout flow. Under heavy load, checkouts submit email requests to SQS, which standardizes delivery and executes automatic retries.

# DynamoDB Chat History Storage Integration
import boto3
from botocore.exceptions import ClientError
from datetime import datetime, timedelta

class DynamoDBSessionService:
    def __init__(self, table_name="blissful-chatbot-history"):
        self.db = boto3.resource('dynamodb', region_name='ap-south-1')
        self.table = self.db.Table(table_name)

    def save_chat_message(self, user_id, message, sender):
        ttl = int((datetime.utcnow() + timedelta(days=30)).timestamp())
        try:
            self.table.put_item(
                Item={
                    'user_id': str(user_id),
                    'timestamp': datetime.utcnow().isoformat(),
                    'message': message,
                    'sender': sender,
                    'ttl': ttl
                }
            )
            return True
        except ClientError as e:
            print(f"DynamoDB Error: {e.response['Error']['Message']}")
            return False

Engineering Problems

Architectural Challenges

Monolithic Session Bottlenecks

Problem

Local Flask session state prevented horizontal application scaling across multiple EC2 server targets.

Solution

Offloaded session storage to a centralized DynamoDB table using TTL variables to clean up expired sessions.

Outcome

Achieved completely stateless application servers, enabling containerized horizontal auto-scaling.

Stateless App Server Migrated

AI Inference Latency

Problem

Evaluating five ML models synchronously during checkout caused slow booking confirmations.

Solution

Migrated high-latency risk calculations to background worker processes triggered asynchronously.

Outcome

Reduced critical API response time from 1.5 seconds down to single-digit millisecond latency.

Non-Blocking Flow Achieved

Security Credentials Exposure

Problem

Developers committing hardcoded IAM API credentials to public repositories created severe security risks.

Solution

Replaced secret keys with IAM Role Profiles attached to the EC2 instances, generating STS temporary keys.

Outcome

Eradicated credentials leakage, aligning application code with security industry standards.

STS Role Permissions Integrated
